# Proposal: Удаление приоритета желаний

## Intent
В текущей реализации желания имеют три уровня приоритета (обычный / высокий / очень высокий), которые влияют на вероятность выпадения в рулетке через weighted random. Пользовательский фидбек показывает, что это создаёт неравенство шансов и лишнюю сложность. Все желания должны иметь равный шанс на выпадение.

## Scope

In scope:
- Удаление поля `priority` из модели `Wish` и базы данных
- Удаление enum `WishPriority` и UI-компонента `PriorityPicker`
- Переход рулетки с weighted random на равновероятный выбор
- Удаление priority из `WishSnapshot`, `WishEditDraft`, бэкапа
- Обновление тестов `WishPickerTest` и `BackupCodecTest`
- Room-миграция БД с удалением колонки (или пересозданием таблицы)

Out of scope:
- Изменение логики фильтрации по бюджету
- Изменение порядка сортировки активного списка
- Изменение UI/UX рулетки за пределами удаления приоритета

## Approach
Последовательное удаление поля из всех слоёв: data → domain → UI. Миграция БД — через пересоздание таблицы с сохранением данных (ALTER TABLE DROP COLUMN не поддерживается SQLite < 3.35, minSdk 26).
