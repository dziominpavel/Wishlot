# Документация Wishlot

**Wishlot** — Android-приложение для личного вишлиста: пользователь копит идеи покупок с ценами, а в момент «побаловать себя» на выбранную сумму приложение **фильтрует** подходящие желания и **случайно** предлагает одно — с анимацией «колеса фортуны» и явным согласием перед отметкой «исполнено».

> Дата ревизии: 2026-05-18 · Код ещё не реализован — документы описывают целевое состояние.

## 🎯 Процесс: Spec-Driven Development (SDD)

Спецификация — **источник истины** для кода. Каждое изменение проходит через этапы:

1. **Спецификация** — AI правит/дополняет спеку (`docs/`).
2. **Согласование** — создаётся `docs/changes/pending/<feature>-<описание>.md`, пользователь читает и апрувит.
3. **Применение** — изменения вносятся в основную спеку, pending-файл удаляется.
4. **Код** — AI пишет код строго по спеке.
5. **Тесты** — AI обновляет тесты и проверяет прохождение.

> **Правило:** код правится **только после** спеки. Никакого кода до апрува.

## Принципы

- Источник истины — **спецификация в `docs/`**; код реализуется только после согласования.
- **GymProgress** и **VoiceMind** — референс **структуры** Gradle, docs, стиля кода (Compose, Room, один ViewModel), не продуктовых требований.
- **`minSdk = 26`** (Android 8.0+) — шире, чем GymProgress (29); как в VoiceMind.
- Планы сессий в `.cursor/plans/`, roadmap — в `FEATURE_PLAN.md`.
