# Wheel Specification

## Purpose
Механика колеса фортуны и логика выбора победителя по бюджету.

## Requirements

### Requirement: Выбор победителя до анимации
Победитель SHALL определяться до запуска анимации колеса.

#### Scenario: Запуск рулетки
- GIVEN активный вишлист не пуст
- WHEN пользователь запускает колесо
- THEN алгоритм выбирает победителя
- AND анимация отображает этот результат

### Requirement: Фильтр по бюджету
Рулетка SHALL учитывать только желания, входящие в заданный бюджет.

#### Scenario: Фильтр по доступной сумме
- GIVEN бюджет рулетки = 5000
- WHEN запускается выбор
- THEN в выборку попадают только желания с budget ≤ 5000

### Requirement: Равновероятный выбор
Все кандидаты на розыгрыш SHALL иметь равную вероятность быть выбранным.

#### Scenario: Равные шансы
- GIVEN три активных желания в рамках бюджета
- WHEN запускается рулетка
- THEN каждое желание имеет одинаковую вероятность 1/3

### Requirement: Random не меняет статус
Процесс случайного выбора SHALL NOT менять статус желания.

#### Scenario: Просто крутим
- GIVEN активное желание участвует в рулетке
- WHEN оно выпадает победителем
- THEN статус остаётся active до нажатия "Беру!"

### Requirement: Слои анимации независимы
Анимация вращения диска SHALL NOT влиять на указатель и центральный элемент.

#### Scenario: Крутится только диск
- GIVEN запущена анимация колеса
- WHEN диск вращается
- THEN указатель остаётся вверху неподвижным
- AND центральная крышка с иконкой подарка остаётся неподвижной
