# UI Specification

## Purpose
Общие правила UI/UX проекта Wishlot.

## Requirements

### Requirement: Язык интерфейса
Весь пользовательский интерфейс SHALL быть на русском языке.

#### Scenario: Открытие любого экрана
- GIVEN пользователь открывает приложение
- THEN все метки, кнопки и подсказки на русском

### Requirement: Тема оформления
Приложение SHALL использовать дизайн-токены из `WishlotTheme` (палитра GIFT SPIN).

#### Scenario: Рендеринг компонентов
- GIVEN экран отрисовывается
- THEN цвета, типографика и формы соответствуют `WishlotTheme`
- AND не используются токены из других тем (IRON CORE, CLEAR BELL)

### Requirement: Отображение цен
Цены SHALL отображаться как число с разделителем тысяч (без символа валюты), с учётом того, что в БД хранятся в minor units.

#### Scenario: Отображение бюджета
- GIVEN в БД budget = 500000 (minor units)
- WHEN отображается на экране
- THEN пользователь видит "5 000"

### Requirement: Отсутствие стрелок сортировки
Карточка желания SHALL NOT содержать кнопок "вверх"/"вниз" для ручной сортировки.

#### Scenario: Просмотр списка
- GIVEN пользователь на экране вишлиста
- THEN на каждой карточке отсутствуют кнопки сортировки

### Requirement: Отсутствие настройки анимации
Экран настроек SHALL NOT содержать переключателя анимации колеса.

#### Scenario: Открытие настроек
- GIVEN пользователь на экране настроек
- THEN отсутствует блок "Анимация колеса"

### Requirement: Экран настроек
Экран настроек SHALL отображать статистику по желаниям, кнопку открытия Истории и версию приложения.

#### Scenario: Просмотр настроек
- GIVEN пользователь на экране настроек
- THEN видна статистика (активных, исполненных, в архиве)
- AND есть кнопка открытия Истории
- AND отображается версия приложения

### Requirement: Навигация — 3 вкладки
Нижняя навигация SHALL содержать ровно 3 вкладки: Вишлист, Крутить, Настройки.

#### Scenario: Открытие приложения
- GIVEN пользователь открывает приложение
- THEN в нижней навигации 3 вкладки
- AND центральная вкладка "Крутить" визуально выделена

### Requirement: История — overlay
Экран Истории SHALL открываться как overlay из экрана Настроек, а не как отдельная вкладка.

#### Scenario: Просмотр истории из настроек
- GIVEN пользователь на экране настроек
- WHEN нажимает "История"
- THEN открывается overlay экрана Истории с TopAppBar и кнопкой "Назад"
