# Wishlist Specification

## Purpose
Управление списком желаний пользователя: создание, редактирование, просмотр и перевод в статус fulfilled.

## Requirements

### Requirement: Создание желания
Пользователь SHALL иметь возможность создать желание с названием, описанием, бюджетом, ссылкой и категорией.

#### Scenario: Успешное создание
- GIVEN пользователь на экране создания
- WHEN он заполняет название и бюджет
- THEN желание сохраняется в активном статусе
- AND оно появляется в списке активных желаний

### Requirement: Отсутствие приоритета
Система SHALL NOT предоставлять пользователю выбор или изменение приоритета желания.

#### Scenario: Создание желания
- GIVEN пользователь на экране создания
- THEN отсутствует поле или переключатель приоритета

### Requirement: Сортировка активного списка
Активный вишлист SHALL отображаться в порядке `createdAt ASC, id ASC`.

#### Scenario: Открытие списка
- GIVEN есть несколько желаний разных дат создания
- WHEN пользователь открывает активный вишлист
- THEN желания отсортированы по возрастанию createdAt, затем id

### Requirement: Статус fulfilled
Желание SHALL переходить в статус fulfilled только после явного подтверждения пользователя ("Беру!").

#### Scenario: Подтверждение после рулетки
- GIVEN рулетка выбрала победителя
- WHEN пользователь нажимает "Беру!"
- THEN статус желания меняется на fulfilled
- AND `fulfilledAt` устанавливается в текущее время

### Requirement: Сортировка истории
История fulfilled SHALL отображаться в порядке `fulfilledAt DESC, id DESC`.

#### Scenario: Просмотр истории
- GIVEN есть выполненные желания
- WHEN пользователь открывает историю
- THEN желания отсортированы по убыванию fulfilledAt
